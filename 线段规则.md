# 线段划分规则

本文记录当前程序实际采用的同级别线段划分规则。线段只基于已确认笔构造，采用单一运行段状态机；任意时刻最多只有最后一段处于运行中。

## 1. 输入条件

1. 输入必须是按时间排列、方向交替且首尾连续的 `CONFIRMED` 笔。
2. `PENDING` 笔不参与线段形成或线段终结确认。
3. 少于 3 笔时不能形成线段。

## 2. 初始线段形成

从左到右寻找第一组连续 3 笔，计算：

- `ZG = min(三笔各自的最高价)`；
- `ZD = max(三笔各自的最低价)`。

只有 `ZG > ZD` 时三笔存在严格公共重叠，线段成立。`ZG = ZD` 仅为边界接触，不视为有效重叠。

线段方向由第一笔决定：

- 第一笔向上，线段为向上线段；
- 第一笔向下，线段为向下线段。

第一组满足条件的三笔作为该线段的形成证据。更早但未形成三笔公共重叠的笔不归入线段。

## 3. 运行段延伸

1. 线段形成后先标记为 `IS_RUNNING`。
2. 未满足终结条件时，后续笔持续归入当前运行段。
3. 向上线段终点取段内向上方向的最高极值；向下线段终点取段内向下方向的最低极值。
4. 相同极值多次出现时，终点取最后一次出现的位置，避免图形停在较早的同价拐点。
5. 普通反向摆动或仅出现一笔反向笔不能终结当前线段。

## 4. 反向线段候选

反向候选必须同时满足：

1. 候选第一笔方向与当前线段方向相反；
2. 候选从当前线段形成三笔之后开始；
3. 候选最初连续 3 笔具有严格公共重叠，即 `ZG > ZD`。

候选形成后可以继续吸收后续笔并更新反向极值，但在满足终结条件前不能提前切断旧线段。

若候选尚未确认时，原线段方向的笔重新突破候选第一笔的原方向极值，则该候选失效：

- 当前为向上线段：后续向上笔最高价高于候选第一笔最高价；
- 当前为向下线段：后续向下笔最低价低于候选第一笔最低价。

失效后继续向右寻找新的反向三笔候选。

## 5. 特征序列确认

特征序列由当前线段内所有反向笔按时间顺序组成，并包含初始成段三笔中的反向笔。初始反向笔可作为第一个有效分型的左侧元素，但新反向线段仍必须在初始三笔之后独立形成。

取最新三个特征序列元素 `A、B、C`：

- 向上线段只观察向下笔。若 `B.high > A.high`、`B.high > C.high`，同时 `B.low > A.low`、`B.low > C.low`，则形成顶分型；
- 向下线段只观察向上笔。若 `B.high < A.high`、`B.high < C.high`，同时 `B.low < A.low`、`B.low < C.low`，则形成底分型。

三个元素连续创新高、创新低或单调下降、上升都不等同于分型，不能据此确认线段终结。

特征序列分型只有在反向三笔候选已经形成有效公共重叠后，才能确认旧线段结束。

## 6. 固定起点边界确认

除特征序列分型外，完整反向线段越过旧线段的固定起点边界也可确认终结：

- 向上线段：反向候选最低价低于旧线段起点价格；
- 向下线段：反向候选最高价高于旧线段起点价格。

该边界在旧线段形成时锁定，后续延伸不会随着段内高低点移动。边界突破仍要求反向三笔候选已经形成，单笔突破不确认线段终结。

## 7. 旧段确认与新段衔接

满足“特征序列分型”或“固定起点边界突破”任一条件后：

1. 旧线段在反向候选开始前锁定，状态改为 `CONFIRMED`。
2. 反向候选成为新的 `IS_RUNNING` 线段。
3. 新线段在显示和结构上锚定到旧线段的真实终点：新段起点索引、时间和价格必须分别等于旧段终点索引、时间和价格。
4. 已确认线段之后只能存在下一条连续线段；结果列表最后始终保留且仅保留一条运行中线段。

## 8. 证据与输出

每条线段至少输出：编号、方向、起止索引、起止时间、起止价格、最高价、最低价、包含笔编号和状态。

同时保留可审计证据：

- 初始成段三笔及其 `ZD/ZG`；
- 反向候选三笔及其 `ZD/ZG`；
- 触发确认的特征序列笔编号和分型类型；
- 固定边界方向及价格；
- 反向候选极值；
- 确认笔、确认时间和确认原因。

状态含义：

- `CONFIRMED`：已经被满足规则的完整反向线段确认终结；
- `IS_RUNNING`：尚未满足终结条件，终点会随新笔继续更新。
