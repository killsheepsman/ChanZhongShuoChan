缠论完全分类与买卖点判定规则库（开发版）
版本：v1.0
适用：股票/期货/数字货币的K线数据自动分类与买卖点识别
依赖：原始K线数据（Open/High/Low/Close/Volume）

一、K线包含关系处理（原始数据清洗）
规则编号	输入数据	判定逻辑	输出结果	编码要点
1-1	连续两根K线（T1, T2）	若 T1.high ≥ T2.high 且 T1.low ≤ T2.low（或反向），即为包含关系	标记为"包含"	需逐根遍历，合并后替换原序列
1-2	包含方向判断	若包含前为上涨（前一根为阳），则取两根中 high 较高者、low 较高者	合并K线（高高/低低）	若前一根为下跌，则取 low 较低者、high 较低者
1-3	合并终止条件	合并后的K线与下一根不再有包含关系	输出标准化K线序列	循环处理直到无包含
二、分型识别（顶/底边界标记）
规则编号	输入数据	判定逻辑	输出结果	编码要点
2-1	连续3根处理后的K线（K1,K2,K3）	K2.high > K1.high 且 K2.high > K3.high	标记为"顶分型"	顶分型出现后，记录 high 值为顶点
2-2	连续3根处理后的K线（K1,K2,K3）	K2.low < K1.low 且 K2.low < K3.low	标记为"底分型"	底分型出现后，记录 low 值为底点
2-3	分型有效性过滤	相邻顶底分型之间，至少间隔1根独立K线（不共用K线）	有效分型序列	可配置参数 MIN_GAP=1（根）
三、笔的划分（最小趋势单元）
规则编号	输入数据	判定逻辑	输出结果	编码要点
3-1	有效分型序列（顶/底交替出现）	底→顶，且顶.high > 底.low + 阈值	向上笔（起点底，终点顶）	阈值建议用 ATR 或最小波动单位（如0.01元）
3-2	有效分型序列（顶/底交替出现）	顶→底，且底.low < 顶.high - 阈值	向下笔（起点顶，终点底）	同上阈值设置
3-3	笔的破坏条件	新笔的起点低于上一笔的终点（上升笔中）或反之	旧笔终止，新笔开始	关键判定：是否突破前一笔的起点
3-4	无效笔过滤	笔的幅度 < 阈值（如最小波动×5）或包含不足K线	丢弃，不纳入后续中枢计算	幅度阈值可配置
四、线段划分（笔的递归组合）
规则编号	输入数据	判定逻辑	输出结果	编码要点
4-1	连续3笔（P1,P2,P3）	P1与P2重叠区域 ∩ P2与P3重叠区域 ≠ 空集	构成一线段	重叠区间 = [min(高点), max(低点)]
4-2	线段方向判定	若线段起始笔为向上笔，则整条线段为"向上线段"	方向标记 (+1 / -1)	线段方向由第一笔方向决定
4-3	线段终结（破坏）条件	反向线段已形成，且反向笔突破前线段终点	旧线段结束，新线段开始	采用特征序列法：反向特征序列出现分型
五、走势类型完全分类（最终输出）
规则编号	输入数据	判定逻辑	输出结果	编码要点
5-1	线段序列	连续3段线段有重叠区域 → 定义为一个中枢	识别中枢，记录 ZG（高点）和 ZD（低点）	重叠区间取各线段高点的最低值、低点的最高值
5-2	中枢数量统计	统计同一方向上已形成的中枢数量 n	n = 0 → 未分类；n = 1 → 盘整；n ≥ 2 → 趋势	向上和向下趋势分别计数
5-3	趋势方向判定	若后续中枢的 ZD > 前中枢的 ZG，则为"上涨趋势"	输出：上涨趋势	若 ZG < 前中枢的 ZD，则为"下跌趋势"
5-4	离开段力度比较	比较进入中枢的线段幅度 vs 离开中枢的线段幅度	若离开段幅度/斜率/MACD面积 < 进入段 → 背驰	此条为买卖点预警信号
5-5	第三类买卖点判定	离开段后，回抽段不进入中枢区间（不触及 ZG 或 ZD）	回抽低点 > ZG → 三买；回抽高点 < ZD → 三卖	买卖点标记
六、三类买点判定规则（做多信号）
规则编号	买点类型	前置条件	精确位置判定	确认信号（必须同时满足）	强度评级因子
6-1	第一类买点（一买）	① 当前走势为下跌趋势（≥2个下跌中枢）
② 处于最后一个中枢的下方	离开段（b段）的最低点	① b段与a段（进入段）发生底背驰（MACD绿柱面积减少 或 黄白线底背离）
② 底分型确认（次级别回抽不破该低点）	背驰越明显越强；
若同时放量，强度+1
6-2	第二类买点（二买）	① 一买已经确认发生
② 当前处于一买后的向上反弹段	反弹后次级别回跌的最低点	① 该回跌低点 ≥ 一买的最低点（不创新低）
② 次级别出现底分型或背驰止跌	越接近一买低点越强（"强二买"）；
若反弹突破前中枢ZD后再回踩，强度+1
6-3	第三类买点（三买）	① 当前走势为上涨趋势或盘整后向上突破
② 已形成一个中枢	向上离开中枢后，次级别回抽的最低点	① 该回抽低点 > 中枢高点（ZG）——绝不回中枢
② 次级别回抽段出现底背驰或底分型止跌	离开中枢力度越强（放量长阳），三买越可靠；
回抽缩量更佳
七、三类卖点判定规则（做空/离场信号）
规则编号	卖点类型	前置条件	精确位置判定	确认信号（必须同时满足）	强度评级因子
7-1	第一类卖点（一卖）	① 当前走势为上涨趋势（≥2个上涨中枢）
② 处于最后一个中枢的上方	离开段（b段）的最高点	① b段与a段（进入段）发生顶背驰（MACD红柱面积减少 或 黄白线顶背离）
② 顶分型确认（次级别回抽不破该高点）	背驰越明显越强；
若放量滞涨，强度+1
7-2	第二类卖点（二卖）	① 一卖已经确认发生
② 当前处于一卖后的向下回调段	回调后次级别反弹的最高点	① 该反弹高点 ≤ 一卖的最高点（不创新高）
② 次级别出现顶分型或背驰滞涨	越接近一卖高点越强（"强二卖"）；
若回调跌破前中枢ZG后再反弹，强度+1
7-3	第三类卖点（三卖）	① 当前走势为下跌趋势或盘整后向下突破
② 已形成一个中枢	向下离开中枢后，次级别回抽的最高点	① 该回抽高点 < 中枢低点（ZD）——绝不回中枢
② 次级别回抽段出现顶背驰或顶分型滞涨	离开中枢力度越强（放量长阴），三卖越可靠；
回抽缩量更佳
八、买卖点过滤与优先级规则（防误判）
规则编号	过滤维度	判定逻辑	输出动作	编码要点
8-1	级别过滤	买卖点必须与当前操作级别一致。
例：日线三买必须在日线图上识别，不能在30分钟图上识别后当作日线买卖点	若级别不匹配 → 标记为"次级别信号"，不触发操作	在工具中预设"主操作级别"参数
8-2	背驰共振过滤	若本级别出现买点，且次级别也同时出现同向背驰 → 信号增强	共振时标记为"★ 强信号"	需递归检查次级别走势
8-3	买卖点互斥过滤	同一位置不能既是买点又是卖点（如盘整背驰后反弹未破中枢，只视为震荡，不产生三类买卖点）	需按优先级：三买/三卖 > 二买/二卖 > 一买/一卖	用状态机管理当前走势归属
8-4	未完成走势过滤	若离开段尚未结束（次级别未出现反向分型），则买卖点处于"待确认"状态，不输出最终信号	标记为 PENDING，等待下一根K线确认	设置确认K线数量（如至少1根反向K线确认）
8-5	成交量辅助过滤（可选）	买点区域：缩量下跌后放量企稳 → 增强
卖点区域：缩量上涨后放量滞涨 → 增强	仅作为辅助权重因子，不单独作为判定条件	使用 VOL_MA_RATIO（当前量/5日均量）

每个被识别出的买卖点，工具应输出以下结构化字段：

字段名	类型	说明	示例值
signal_type	string	买点/卖点类型	BUY_1 / BUY_2 / BUY_3 / SELL_1 / SELL_2 / SELL_3
occur_time	datetime	信号出现的时间戳（K线收盘时间）	2026-07-07 14:55:00

level	string	所属级别	日线 / 30分钟 / 5分钟
strength	int	强度评分（1~5）	4
status	string	状态	CONFIRMED / PENDING / INVALIDATED
relate_center_high	float	关联中枢高点（ZG）	16.20
relate_center_low	float	关联中枢低点（ZD）	15.50
backup_ratio	float	背驰幅度（进入段 vs 离开段力度比）	0.72（表示离开段力度为进入段的72%）
risk_warning	string	风险提示	次级别未背驰，谨慎 / 量价配合良好

十一、编码处理优先级（状态机流程）
每根新K线到来时，按以下顺序处理：

text
1. 更新K线数据 → 处理包含关系（表1）
2. 识别分型（表2）→ 划分笔（表3）
3. 组合线段（表4）→ 识别中枢（表5）
4. 判断当前是否处于"离开段"中
   ├── 若否 → 标记为震荡，不产生买卖点
   └── 若是 → 比较离开段与进入段力度
5. 是否背驰？
   ├── 若背驰 → 标记一买/一卖（PENDING）
   │   └── 等待次级别反向分型确认 → CONFIRMED
   │       ├── 回抽不破前低/前高 → 二买/二卖
   │       └── 回抽不触中枢 → 三买/三卖
   └── 若不背驰 → 继续等待
6. 若回抽重新进入中枢 → 之前买卖点作废（INVALIDATED）
十二、开发避坑指南
序号	问题点	建议解决方案
1	包含关系合并时浮点数精度问题	设定 TOLERANCE = 1e-6，比较时用 abs(a-b) <= TOLERANCE
2	递归处理（笔→线段→中枢）性能消耗大	使用流水线架构，每层输出缓存，增量更新而非全量重算
3	实时数据最后一笔/段未结束	标记为 PENDING 状态，设计"延后确认"机制
4	不同时间级别信号冲突	设定"主操作级别"，高级别定方向，低级别定入场点
5	盘整中反复出现买卖点假信号	严格限制"三类买卖点"必须在离开段产生，盘整内只做震荡处理
