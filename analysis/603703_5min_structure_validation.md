## Important data limitation

Evidence: 205 of 1207 exported rows have an open or close outside the stated high-low range. That can occur after inclusion processing because the merged bar retains the original open and final close while changing high-low boundaries. It is not valid raw market OHLC.

Therefore this file can validate the already-processed high-low sequence, fractals, strokes, and later structures. It cannot independently validate the original inclusion step. The API now returns `raw_klines`, and CSV export now uses that field so a fresh export can validate the full raw-K-line-to-inclusion chain.

## Corrections made

1. Fractal gap: corrected the minimum distance so adjacent three-bar fractal windows neither share a K-line nor omit the required independent bar.
2. Stroke length: raised the default endpoint distance from 3 to 4 processed indices, which means at least 5 K-lines including endpoints.
3. Segment state machine: a reverse overlap must now satisfy both conditions: form a reverse three-stroke unit and break the current segment boundary. The current running segment is no longer confirmed merely because a reverse overlap exists.
4. Segment continuity: the new segment is anchored at the prior confirmed segment endpoint at the state transition itself. The old code only changed its visible start after detection, which made the chart look continuous while the boundary logic used a different start point.
5. Segment range retention: anchoring no longer discards the candidate unit high/low range. This prevents incorrect ZD/ZG calculations.
6. Pending stroke: the most recent unfinished bar is no longer appended as a confirmed stroke before segment, center, divergence, and signal analysis.
7. CSV export: raw input K-lines are returned separately and exported instead of inclusion-processed chart bars.
